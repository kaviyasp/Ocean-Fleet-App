# OceanFleet App

## Project Overview
OceanFleet App is a Java console-based application developed to demonstrate Object-Oriented Programming (OOPS) concepts and Java Collections through a real-world vessel performance tracking scenario.

---

> **Note:**  
> The complete source code for this project is available in the `dev` branch.  
> The `main` branch contains only project overview and documentation.

---

# Objectives

- Store vessel performance details
- Retrieve vessel information using vessel ID
- Identify high-performance vessels based on speed
- Apply clean OOPS design principles

---

# Use Cases Implemented

## UC1: Create Vessel Model

### Objective
Create a vessel model class to represent vessel details.

### Description
This use case defines the structure of a vessel object using Object-Oriented Programming concepts.  
The vessel model stores information such as vessel ID, vessel name, speed, and performance details.

### Outcome
- Vessel class is created successfully
- Vessel properties are properly defined
- OOPS principles are applied

---

## UC2: Store Vessel Records using List

### Objective
Store multiple vessel records dynamically.

### Description
This use case uses Java Collections (`List`) to manage vessel records efficiently.  
It allows the application to store and organize multiple vessel objects during runtime.

### Outcome
- Vessel records are stored dynamically
- Multiple vessel entries are supported
- Collection framework usage is implemented

---

## UC3: Retrieve Vessel by Vessel ID

### Objective
Retrieve vessel details using vessel ID.

### Description
This use case searches for a vessel record based on the unique vessel ID entered by the user.

### Outcome
- Vessel details are retrieved successfully
- Search functionality is implemented
- Accurate vessel identification is ensured

---

## UC4: Identify High Performance Vessels

### Objective
Identify vessels with high performance based on speed.

### Description
This use case evaluates vessel performance using speed values and filters vessels that meet high-performance conditions.

### Outcome
- High-performance vessels are identified
- Speed-based filtering is implemented
- Performance analysis is supported

---

## UC5: Console-Based User Interaction

### Objective
Provide user interaction through the console.

### Description
This use case enables users to interact with the system using menu-driven console operations for adding, retrieving, and viewing vessel records.

### Outcome
- Interactive console interface is provided
- User inputs are accepted dynamically
- System navigation becomes user-friendly

---

## UC6: Refactor using OOPS Principles

### Objective
Improve application structure using OOPS concepts.

### Description
This use case refactors the application to follow clean coding standards and Object-Oriented Programming principles such as encapsulation, modularity, and abstraction.

### Outcome
- Code structure becomes modular
- Application maintainability improves
- Reusability and readability are enhanced
